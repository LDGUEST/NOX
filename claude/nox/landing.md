Start drafting a landing page using the project's existing stack. If no framework is detected, default to Next.js App Router + TypeScript + Tailwind CSS.

**Adapt to the project's tech** — if it uses Vite, Astro, SvelteKit, or another framework, use that instead.

## Process

1. **Wireframe** — Create a layout structure: hero, features, social proof, CTA, footer
2. **Copy** — Write conversion-focused copy for each section. Focus on benefits, not features.
3. **Components** — Generate the page as composable components (keep each under 200 lines)
4. **Hero Graphic** — Add an animated SVG or illustration for the hero section
5. **Responsive** — Ensure the layout works on mobile, tablet, and desktop

## Guidelines

- Use functional components and hooks only
- Keep files under 200 lines — split into sections if needed
- Follow the project's existing styling patterns
- Include semantic HTML for accessibility
- Add meta tags for SEO (title, description, og:image)
